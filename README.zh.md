## 简介

Sovereign是一款**基于“汉密尔顿系统”的“大富翁”风格棋盘游戏**，讲述的是美国公共信贷的建立，同时还提供一个**完整的单人/数字版本**，它可以在浏览器中本地运行，与两个具有固定行为的AI对手对战。

- **桌面游戏** — 可打印的 34 页版本。 棋盘包含 40 个格子，22 个地产 + 4 条路线 + 2 个机构，8 个颜色系统，7 项国会法案按照固定的历史顺序排列，4 个玩家角色，3 条共享进度条（公共信用 · 公共抵抗 · 工业产能），12+12 张事件卡牌。 除了财政部之外，还有两种可行的经济发展路径：商人路线和制造商路线。 v0.2 版本，已冻结。
- **数字模式** — 单个独立的 HTML 文件。 基于电路的结束条件：当一名玩家完成第四次“财政部开放”时，游戏结束。 游戏平均时长约为 23 轮（67 回合）。 在最终结算时，影响力最高的玩家获胜，*不一定是最先绕完棋盘的玩家*。 战略深度层：三个锁定角色的特殊行动，六张带有时机窗口的 HAND 卡牌，改革恢复行动，多阶段的信用压力（公众怀疑/危机/恐慌/违约）。 战略弧线层：从第 8 轮开始，每轮触发八个联邦时代事件，三个带有游戏结束奖励的个人视角。 编年史家叙述层：14 个与事件相关的历史提示，来自《联邦主义论文集》和 Founders Online 的真实引言，带有可关闭的持续提示。 使用确定性的 mulberry32 随机数生成器，内置脚本 AI 对手，支持保存/加载，并具有哈希完整性校验，支持回放功能，以及设计师专用的批量模拟工具。
- **平衡基准** — 电路 + 战略深度 + 战略弧线 + 编年史家 (v1.4.0 beta)：财政部 59% · 商人 20% · 制造商 21% (标准值 × 100)。 三种角色发展路径都具有显著的获胜机会。 个人视角达成率：联邦信用架构师 54%，商业主权者 39%，工业先驱 29%。 基础的 v0.18 机制（信用危机、现金知识产权计分、工业特许、完成套装的奖励）与 v0.3 → v0.10 → v0.18 的设计版本完全一致，并通过 1000 多个确定性模拟游戏进行验证。

## 快速开始

### 在浏览器中直接玩（无需安装）

```bash
npx @mcptoolshop/sovereign
```

CLI命令可以打开游戏，使用您的默认浏览器。 无需安装，无需服务器，无需连接互联网。

其他模式：

```bash
npx @mcptoolshop/sovereign --print    # Open the printable 34-sheet board game
npx @mcptoolshop/sovereign --start    # Open the audience-routed landing page
npx @mcptoolshop/sovereign --path     # Print the playable HTML file path
npx @mcptoolshop/sovereign --help     # All flags
```

### 在线游戏

打开位于 **<https://mcp-tool-shop-org.github.io/sovereign/>** 的主页，然后点击进入数字游戏。

### 打印并玩

可打印的棋盘游戏是一个独立的34页HTML文档。 打开 `release/board-game/sovereign-board-game.html` 文件（或从下载的文件中），然后使用 `Cmd/Ctrl-P → 保存为PDF → US Letter → 100% 比例`。 剪切并开始游戏。

### 离线发布包

每个版本都会在GitHub发布页面上附加一个 `sovereign-vX.Y.Z-release.zip` 的发布包。 下载它，解压缩，然后打开 `00-START-HERE.html` 文件，该文件会引导您进入游戏。 整个游戏可以在离线状态下运行。

## 设计理念

Sovereign 的核心思想是，**公共信贷和联邦财政**是亚历山大·汉密尔顿最重要的经济杠杆——但一款“汉密尔顿系统”游戏必须允许**商业**和**工业**也成为通往胜利的可行途径。 从 v0.2 到 v0.10 的平衡迭代是一个持续九个版本的、以证据为导向的过程，旨在保持财政部作为最强大的模式（符合历史），同时避免游戏设计陷入单一策略。

请查看 [`CHANGELOG.md`](./CHANGELOG.md) 文件，了解每个版本的详细变化。

## 确定性

相同的种子 + 相同的玩家决策 = 跨运行、浏览器和操作系统，账本完全一致。

- 单个随机数生成器：`mulberry32(state.rngSeed)`。
- AI 决策：是基于可见状态的纯函数，并且每个决策都会记录到账本中，并附带触发规则。
- 保存/加载功能会保留状态哈希。
- 回放功能可以从 `initialState(seed) + decisionLog` 重建游戏。
- 在 v0.2 到 v0.10 的平衡迭代过程中，经过了 1000 多次确定性游戏的验证。

## 威胁模型与数据处理

Sovereign 是一款独立的、基于浏览器的棋盘游戏。命令行界面会在您的默认浏览器中打开一个本地 HTML 文件。它没有服务器，没有网络调用，没有账户，也没有云同步。

- **涉及的数据：** `release/` 目录中的 HTML 文件（只读），以及 `sovereign.autosave` 键下的 `localStorage`（仅用于游戏存档状态）。
- **未涉及的数据：** 不会访问包目录之外的文件系统，不会进行任何类型的网络请求，不会收集任何遥测数据，不会进行任何分析，不会存储任何凭据。
- **所需权限：** 能够启动操作系统默认浏览器，能够读取包自身的配置文件，以及可选的浏览器 `localStorage` 访问权限。
- **绝不收集任何遥测数据。** 模拟器的“遥测”功能指的是从浏览器中的账本中提取的本地游戏分析报告，这些报告永远不会离开您的设备。

请参阅 [`SECURITY.md`](./SECURITY.md) 以获取漏洞报告和完整的安全策略。

## 功能

- **单人游戏模式：** 对抗两个预设的对手（默认情况下为财政/金融部门和商人/基础设施部门；制造商/工业部门可用于批量游戏）。游戏结束时，当一名玩家完成第四次“财政开放”时，游戏结束。平均游戏时长约为23轮/67步。最终结算时，影响力最高的玩家获胜。
- **战略深度（v1.2版本）：** 三个具有固定属性的特殊行动（发行联邦债券/经纪合同/开设作坊），6张手牌，带有时间窗口（手牌上限为2张），恢复行动，多阶段的信用压力（公众疑虑/危机/恐慌/违约）。
- **战略发展（v1.3版本）：** 每轮从第8轮开始，触发8个联邦时代事件（5个选项 + 3个自动触发），3个具有不同属性的“远景”（联邦信用架构师/商业主权/工业先驱），游戏结束时可获得+3 IP的额外奖励。
- **编年史（v1.4版本）：** 以第三人称叙述的历史声音。14个与事件相关的横幅（法案×7 / 联邦时代开端 / 疑虑 / 危机 / 恐慌 / 违约 / 叛乱 / 改革 / 远景 / 最终结算）。所有引用的内容均已通过founders.archives.gov、Wikisource和国会图书馆等来源进行验证。失败的法案将被描述为与真实历史的“反事实”叙述（“在我们的历史中，汉密尔顿的融资法案于1790年7月以32票对29票通过；在你的共和国中，士兵歧视获得了足够的票数，以阻止其通过。”）。带有防伪涂层的提示框，可选择关闭或设置为最小显示，并尊重叙述的开启/最小/关闭设置。
- **确定性人工智能：** 每一个对手的决策都是对可见状态的纯粹函数，并带有记录的理由。没有大型语言模型，没有不透明的“魔法”。
- **8个游戏界面：** 游戏面板、财政面板、资产检查器、事件抽屉、国会法案、共享轨道、回合记录/账本、游戏结束报告。
- **拍卖：** 未被接受的资产将进入多人拍卖，采用基于属性的预设竞标。
- **保存/加载：** 每回合自动保存到`localStorage`，支持手动JSON导出/导入，加载时进行哈希完整性检查，并根据版本进行限制。
- **回放：** 可以完整回放任何已完成的游戏。只读模式。通过种子和决策日志进行重建，并带有绿色的完整性标识。
- **批量模拟：** 运行10/50/100个确定性游戏，针对任何属性组合，导出JSON和HTML报告，用于平衡性分析。
- **历史叙述：** 从账本中提取的25条记录（默认40-60字，扩展版150-200字，游戏结束时提供约300-500字的共和国总结）。永远不会改变游戏状态。
- **可访问性：** 完整的键盘导航，焦点指示器，屏幕阅读器友好的标签，可以以文本形式读取的数值（而不仅仅是标记），最小字体为14px，尊重减少动画的设置。

## 配置组合 (v0.10)

| 配置 | 资产优先级 | 优势 | 劣势 |
|--------------------------------|---------------------------------------------------------------|----------------------|-------------------------------------|
| **Treasury / Finance**         | NF > 州债务 > 财政债务 > 银行 > 铸币厂 | 公共信用增长 | 无基础设施收入 |
| **Merchant / Infrastructure**  | 路线 (全部 4 条) > 商业 > 改进 > 收入 | 路线等级 | 无工业产能评分 |
| **Manufacturer / Industry**    | 制造业 > 战略工业 > 改进 > 银行 | 产能倍数 | 开局较慢；获得初始特许权 |

第四个概念文档配置（机会主义者/现金）已推迟。当前 v0.10 版本中可用的竞争配置为三个。

## 已知问题

- **v1.4.0版本为测试版。** 每个功能模块（战略深度、战略弧线、编年史家）都经过了结构性审计和验证，并与模拟诊断结果进行对比。完整的四层结构在模块级别上已经进行了初步测试，但尚未由全新的人类玩家进行端到端的测试。数据来自标准配置下的测试：100次测试中，100次触发了结束条件，平均游戏时长为23轮，获胜者比例为59/20/21，每局游戏大约出现6-8个编年史家事件，大约有33次互动。请将其视为可选功能，直到您亲自体验过为止。
- **人工智能（AI）的策略尚未适应v1.2-v1.4版本的机制。** 它们仍然使用v0.18版本的决策功能，这意味着它们不会像人类玩家那样“争夺视野”或“战略性地使用行动卡”。 实际的人类游戏体验与标准配置下的数据可能会有所不同。
- **触发事件≠获胜。** 在大约三分之一的游戏中，完成第四个循环的玩家仅通过影响力获得胜利。 这是一个有意为之的设计——“最终结算”更注重经济深度，而不是在游戏板上的速度。 游戏结束时的提示会明确这一点。
- **晚期共和国阶段很长，且没有新的事件触发。** 事件仍然在1-7轮触发。 平均游戏时长为23轮，这意味着在23轮中，大约有16轮处于“晚期共和国”阶段，没有新的政治事件发生。 如果您觉得这段时间过于单调，下一个改进方向是重新分配事件，而不是恢复“授权”机制。
- **财政/金融系统仍然是设计上最强大的，** 并且在目标范围内。 这符合历史背景：公共信用和联邦财政是汉密尔顿的主要经济手段。
- **失败事件（违约/叛乱）仍然主要起到装饰作用。** 信用危机偶尔会触发；违约和叛乱几乎从不发生。 升级系统有更多时间累积，但仍然很少达到最高等级。 未来的版本可能会重新考虑失败状态的压力。
