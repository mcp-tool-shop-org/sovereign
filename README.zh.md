## 这是什么

《主权》是一款**基于《汉密尔顿体系》的《大富翁》规则的棋盘游戏**，讲述了美国公共信用的建立，以及一个**单人/数字改编版**，它在浏览器中运行相同的规则，与两个确定的脚本对手进行对战。

- **棋盘游戏**——可打印的 34 页版本。40 个格子，22 个地块 + 4 条路线 + 2 个机构，8 种颜色系统，7 个按固定历史顺序排列的法案，4 个玩家角色，3 个共享轨道（公共信用、公共抵抗、工业能力），12+12 张事件卡。除了财政部之外，还有两种可行的经济路径：商人和制造商。v0.2 平衡，已固定。
- **数字模式**——一个独立的 HTML 文件。基于电路的结束条件：当一个玩家完成其第四次穿越财政部时，游戏结束。平均游戏时长为 **~22 轮（~66 步）**。在最终结算时，影响力最高的玩家获胜，*不一定是第一个绕完棋盘的玩家*。竞争对手存在层：可见的影响力排名 + 每个对手的策略线。战略深度层：三个角色锁定的特殊行动，六张具有时间窗口的行动卡，改革恢复行动，多阶段的信用螺旋（公共怀疑 → 危机 → 恐慌 → 违约），以及一种偿还债务的机制，预示着加速走向违约，并提供预测和改革的希望。战略弧层：从第 8 轮开始，每轮触发八个联邦时代事件，三个角色愿景，具有游戏结束时的奖励。编年史叙事层：14 个与事件相关的历史横幅，以及信息丰富的等级 B（“了解更多”弹出窗口，编年史记录百科全书，术语表提示），来自《联邦党人论文》和《创始者在线》的真实引言，以及持续显示的提示，并带有 × 按钮以关闭。视觉效果和声音：数字变化，ZzFX 程序音频（13 种提示音），电影/普通/快速即时速度设置。确定性的 mulberry32 随机数生成器，脚本 AI 对手，具有哈希完整性的保存/加载，重播滑块，设计师控制的批量模拟工具。
- **平衡基准**——电路 + 战略深度 + 战略弧 + 编年史 + 信用螺旋（v1.5.0 测试版）：财政部 **48%** · 商人 **34%** · 制造商 **18%**（规范 × 100，通过 `test/measure-stats.mjs` 测量，与实时引擎进行比较）。所有三种角色都能获得有意义的胜利，财政部是最强的，这与历史主题相符。所有三种角色愿景（联邦信用架构师/商业主权者/工业创始人）都是可实现的，并且大致平衡——每种愿景在大约 41-43% 的游戏中触发。底层 v0.18 机制（信用危机、现金 IP 评分、工业特许权、完成套装奖励）与 v0.3 → v0.10 → v0.18 设计弧完全相同，该设计弧由 1000 多个确定性模拟游戏驱动。

## 快速开始

### 在浏览器中游玩（无需安装）

```bash
npx @mcptoolshop/sovereign
```

CLI 会在默认浏览器中打开游戏。无需安装程序，无需服务器，无需互联网。

其他模式：

```bash
npx @mcptoolshop/sovereign --print    # Open the printable 34-sheet board game
npx @mcptoolshop/sovereign --start    # Open the audience-routed landing page
npx @mcptoolshop/sovereign --path     # Print the playable HTML file path
npx @mcptoolshop/sovereign --help     # All flags
```

### 在线游玩

打开托管的登录页面 **<https://mcp-tool-shop-org.github.io/sovereign/>**，然后点击进入数字游戏。

### 打印并游玩

可打印的棋盘游戏是一个独立的 34 页 HTML 文档。从软件包（或从下载）中打开 `release/board-game/sovereign-board-game.html`，然后执行“命令/控制-P → 另存为 PDF → 美国信纸 → 100% 比例”。剪切并开始游戏。

### 离线发布包

每个带有标签的发布版本都会在其 GitHub 发布页面上附加一个名为 `sovereign-vX.Y.Z-release.zip` 的压缩包。下载它，解压缩，然后打开 `00-START-HERE.html`，即可进入面向用户的起始界面。所有操作均可在离线状态下进行。

## 存在的原因

《主权》的核心观点是，**公共信用 + 联邦财政** 是亚历山大·汉密尔顿的主要经济手段——但汉密尔顿体系下的游戏必须让**商业**和**工业**也成为通往胜利的有效途径。平衡调整（v0.2 → v0.10）是一次为期九个版本的、以证据为驱动的尝试，旨在保持财政部作为最强大的战略优势（与历史一致），同时避免将游戏设计简化为单一战略。

请参阅 [`CHANGELOG.md`](./CHANGELOG.md)，以了解完整的版本演进过程。

## 确定性

相同的种子 + 相同的人工决策 = 在不同运行、浏览器和操作系统上产生完全相同的账本。

- 单一随机数生成器：`mulberry32(state.rngSeed)`。
- 对手决策：是可见状态的纯函数，每个决策及其触发规则都会记录在账本中。
- 保存/加载循环可以保留状态哈希。
- 重放功能可以从 `initialState(seed) + decisionLog` 中重建。
- 在 v0.2 → v0.10 平衡调整期间，已在 1000 多个确定性游戏中进行了验证。

## 威胁模型和数据处理

《主权》是一个独立的基于浏览器的棋盘游戏。命令行界面会在您的默认浏览器中打开一个本地 HTML 文件。没有服务器、没有网络调用、没有帐户、没有云同步。

- **涉及的数据：**包含在 `release/` 目录中的 HTML 文件（只读）以及 `localStorage` 中 `sovereign.autosave` 键下的数据（仅为游戏保存状态）。
- **不涉及的数据：**没有对包目录之外的文件系统进行访问，没有进行任何类型的网络请求，没有遥测数据，没有分析数据，没有凭据。
- **所需权限：**能够启动操作系统默认浏览器，能够读取包自身的文件，浏览器 `localStorage`（可选）。
- **绝不会进行任何形式的遥测。** 模拟器的“遥测”功能是指从浏览器内账本中提取的本地游戏分析报告；这些报告绝不会离开您的设备。

请参阅 [`SECURITY.md`](./SECURITY.md)，以了解漏洞报告和完整的安全策略。

## 功能

- **单人模式胜利游戏**，对抗两个预设的对手（默认情况下为财政部/金融部和商人/基础设施；制造商/工业可用于批量游戏）。当一名玩家完成其第四次“财政部开放”行动时，游戏结束。平均回合数为 22 回合 / 66 步。在最终结算时，影响力最高的玩家获胜。
- **对手存在感（v1.5 版本）**——可见的影响力排名和每个对手的行动姿态线，用于描述每个对手的行动相对于*你*在比赛中的地位（“汉密尔顿——影响力领先 3 点——控制了银行；财政集团变得更加团结。”）。结束了类似单人游戏的体验；对手表现得像真正的对手。仅用于呈现——绝不会写入哈希记录。
- **信用螺旋（v1.5 版本）**——公共信用的失败现在会产生影响，并可以进行修复。在信用较低时，会对债务偿还进行现金征收，预示着加速走向违约，并预测趋势，以及改革行动作为真正的救命稻草。它直接传达了公民理念：你能够感受到*为什么*联邦公共信用如此重要。它整合了 v0.18 版本的层次结构（公众怀疑 → 危机 → 恐慌 → 违约），但不会改变其阈值；应用于 `reduce()` 函数中，以确保游戏的可重复性。
- **特效 + 音效（v1.5 版本）**——数值过渡，具有增益/损失不对称性，13 种音效的 ZzFX 程序音频，动作编排，以及速度设置（电影模式 / 正常模式 / 快速模式——快速模式跳过所有动画，以加快游戏速度并提高可访问性）。全程支持完整的键盘操作 / 减少动画 / 屏幕阅读器。
- **战略深度（v1.2 版本）**——三个受限制的特殊行动（发行联邦债券 / 经纪路线合同 / 成立工作室），6 张具有时间窗口的手牌（手牌上限为 2），改革恢复行动。
- **战略弧线（v1.3 版本）**——从第 8 回合开始，每回合触发 8 个联邦时代事件（5 个选择 + 3 个自动），3 个角色愿景（联邦信用架构师 / 商业主权者 / 工业奠基人），具有 +3 IP 终局奖励。所有三个愿景都是可以实现的（每个愿景的概率约为 41-43%，CANONICAL × 100）。
- **编年史家（v1.4 版本）**——以第三人称叙述的历史声音。14 个与事件相关的横幅（《法案》× 7 / 联邦时代开始 / 怀疑 / 危机 / 恐慌 / 违约 / 叛乱 / 改革 / 愿景 / 最终结算）。所有引用的内容均已与 founders.archives.gov、Wikisource 和国会图书馆的资料进行核对。失败的《法案》被叙述为与真实历史相反的情况（“在我们的历史中，汉密尔顿的《融资法案》于 1790 年 7 月以 32 比 29 的票数通过；在你的共和国中，对士兵的歧视获得了足够的选票，以阻止其通过。”）。持续的带有边框的提示，点击 × 即可关闭；尊重叙述的“开启/最小/关闭”设置。
- **编年史家 B 级——信息层（v1.5）**——15 个关于关键机制的“了解更多”弹出窗口，《编年史家记录》百科全书（27 个经过验证的历史引文，以及《法案》、联邦时代事件、信用等级和愿景，全部在一个参考叠加层中），以及 10 个术语表提示。将时代背景转化为实际的、可浏览的历史层。
- **新手引导（v1.5 版本）**——一个 Swift-Start 引导的“1790 年融资辩论”开场，引导新手玩家进入核心循环，此外还有一个“无隐藏”悬停/焦点提示，在玩家做出行动之前，会显示每个行动的成本和后果。
- **确定性 AI**——每个对手的决策都是可见状态的纯函数，并附带记录在案的理由。没有 LLM，也没有不透明的魔法。
- **8 个游戏界面**——游戏面板、财政部面板、资产检查器、事件抽屉、《国会法案》、共享轨道、回合日志/记录、终局报告。
- **拍卖**——被拒绝的资产将进行多玩家拍卖，并根据角色进行预设的竞标。
- **保存/加载**——在每个回合自动保存到 `localStorage`，手动 JSON 导出/导入，加载时进行哈希完整性检查，并根据版本进行限制。
- **重播**——可以对任何已完成的游戏进行完全的滚动浏览。只读。从种子 + 决策日志重建，并带有绿色的完整性标记。
- **批量模拟**——针对任何角色组合，运行 10 / 50 / 100 个确定性游戏，导出 JSON + HTML 报告以进行平衡分析。
- **历史叙述**——25 个条目的库，来源于记录（40-60 字的默认值，150-200 字的扩展，约 300-500 字的终局共和国摘要）。绝不会改变游戏状态。
- **可访问性**——完整的键盘导航、焦点指示器、屏幕阅读器可识别的标签、轨道值可以作为文本而不是标记进行读取，最小字体为 14 像素，尊重减少动画的设置。

## 已知限制

- **v1.5.0 是一个测试版本，等待进行一次完整的真人流程测试。** 每个环节都经过了结构审计，并通过 `test/measure-stats.mjs` 以及确定性和可玩性测试，与实际运行的引擎进行了重新验证；整个系统已经进行了分层测试，但尚未由真人玩家进行完整的流程测试。该流程测试是公开发布版本的可玩性关卡。以下数据是与实际运行的引擎相比的规范化数据（乘以 100）；实际真人游戏可能会有所不同。在您（或您信任的人）进行测试之前，请将其视为可选功能。
- **失败压力是可以感受到且可以恢复的——不再仅仅是装饰。** “信用危机”现在在约 **29/100** 场游戏中触发，并且可以真正地恢复：在进入危机状态（信用 ≤ 4）的游戏中，大约 **41%** 的游戏能够恢复到稳定的信用 ≥ 7，没有游戏会达到“违约”状态。恐慌的情况很少见（约 1/100）。在脚本化的 v0.18 AI 下，“违约”和“叛乱”仍然很少见——该 AI 会在情况恶化之前进行自我改革——但两者都可以通过一个忽略公共信用的真人玩家来实现。信用螺旋使通往“违约”的趋势变得可见且可感知，而不是突然的断崖。
- **对手使用 v1.2–v1.4 系统；只有核心行动机制是 v0.18。** 脚本化的对手*确实*会使用特殊行动、改革补救措施、联邦时代/后期事件选择、投票以及手牌时机——之前的“AI 不会适应”的说明过于宽泛。仍然是 v0.18 的是**核心购买/拍卖/升级/投票**的评估：它们会根据各自的设定进行最佳选择，但尚未像真人优化者那样明确地“争夺愿景”。规范化数据（乘以 100）反映了脚本化的行为；真人游戏可能会有所不同。
- **触发事件 ≠ 获胜。** 完成第四轮回路的玩家，只有在约三分之一的游戏中才能通过影响力获胜。这是有意的——最终结算会奖励经济深度，而不是在棋盘上快速移动。游戏末期的说明明确了这一点。
- **联邦时代环节的行动压力较小。** 基础行动在第 1-7 轮触发；平均游戏时长约为 22 轮，因此联邦时代主要依靠其自身的事件（从第 8 轮开始，每轮触发一次），以及信用螺旋和愿景竞赛。v1.3 版本中，每轮触发一次的联邦时代抽牌机制，将原本的 4 轮空白期减少到约 2/100。如果经过测试后，仍然觉得该环节不够充分，那么下一步应该是重新分配行动，而不是恢复到之前的设定。
- **财政/金融仍然是有意为之的最强环节**（占 48% 的胜场），符合目标范围。这与历史论点相符：公共信用 + 联邦金融是汉密尔顿的主要经济杠杆——但不会使整个设计沦为单一策略（商人占 34%，制造商占 18%）。

## 构建和贡献

```bash
git clone https://github.com/mcp-tool-shop-org/sovereign
cd sovereign
npm install
npm test           # smoke tests
npm run verify     # full verify (smoke + pack-dry-run + CLI flag check)
npm run play       # open the game locally
```

发布版本通过 GitHub Actions (`release.yml`) 发布到 npm，发布时会进行 Sigstore 来源证明。`main` 分支是源代码的最终版本。
