## 它是什么

《Sovereign》是一款**基于汉密尔顿体系的《大富翁》规则的棋盘游戏**，讲述了美国公共信用的建立，同时也是一款**单人/数字改编版**，它在浏览器中本地运行相同的规则，与两个确定的脚本对手对战。

- **棋盘游戏**——可打印的34页版本。40格棋盘，22个地块+4条路线+2个机构，8个颜色系统，7个按固定历史顺序排列的《国会法案》，4个玩家角色，3个共享轨道（公共信贷·公共抵制·工业能力），12+12张事件卡。除了财政部之外，还有两条可行的经济路径：商人与制造商。v0.2版本，已冻结。
- **数字模式**——一个独立的HTML文件。基于电路的结束条件：当一名玩家完成其第四次穿越财政部时，游戏结束。平均游戏时长**约为22回合（约66次行动）**。在最终结算时，影响力最高的玩家获胜，*不一定是第一个绕完棋盘的玩家*。竞争对手存在层：可见的影响力排名+每个对手的姿态线。战略深度层：三个锁定配置的特殊行动，六张带有时间窗口的“手牌”，改革恢复行动，多阶段的信贷螺旋（公共怀疑→危机→恐慌→违约），以及债务偿还税，预先告知的加速、预测和改革的生命线。战略弧层：从第8回合开始，每个回合触发八个联邦时代事件，三个带有游戏结束奖励的个人愿景。编年史叙事层：14个与事件相关的历史横幅，以及信息丰富的B级内容（了解更多弹出窗口、编年史记录百科全书、术语提示），来自《联邦党人论文》和《开国元勋在线》的真实引言，以及带有“×”关闭按钮的持续提示。视觉和声音效果：数字过渡动画，ZzFX程序音频（13个音效），电影/普通/快速即时速度设置。确定性的mulberry32随机数生成器，脚本AI对手，带有哈希完整性的保存/加载功能，重播功能，以及设计师控制的批量模拟工具。
- **三种通往胜利的真实路径**——财政部、商人、制造商，每种方式都能带来有意义的胜利，其中财政部是最强大的，这与历史相符：公共信贷和联邦金融是汉密尔顿最重要的经济杠杆。每种角色都以不同的方式进行游戏，拥有自己的特殊行动和需要追求的个人愿景。

## 快速开始

### 在您的浏览器中游玩（无需安装）

```bash
npx @mcptoolshop/sovereign
```

CLI（命令行界面）会在您的默认浏览器中打开游戏。无需安装程序、服务器或互联网连接。

其他模式：

```bash
npx @mcptoolshop/sovereign --print    # Open the printable 34-sheet board game
npx @mcptoolshop/sovereign --start    # Open the audience-routed landing page
npx @mcptoolshop/sovereign --path     # Print the playable HTML file path
npx @mcptoolshop/sovereign --help     # All flags
```

### 在线游玩

打开托管的登录页面，网址为**<https://mcp-tool-shop-org.github.io/sovereign/>**，然后点击进入数字游戏。

### 打印并游玩

可打印的棋盘游戏是一个独立的34页HTML文档。从软件包中打开`release/board-game/sovereign-board-game.html`（或从下载中打开），然后执行“Cmd/Ctrl-P → 另存为PDF → 美国信纸 → 100%比例”。剪切并开始游戏。

### 离线发布包

每个标记的发布版本都会在其GitHub发布页面上附加一个`sovereign-vX.Y.Z-release.zip`包。下载它，解压缩，然后打开`00-START-HERE.html`，即可进入面向不同用户的入口点。所有内容均可在离线状态下运行。

## 它存在的理由

《Sovereign》的核心观点是，**公共信贷+联邦金融**是亚历山大·汉密尔顿最重要的经济杠杆——但汉密尔顿体系的游戏必须让**商业**和**工业**也成为可行的胜利路径。平衡弧（v0.2 → v0.10）是一个为期九个版本的、基于证据的尝试，旨在保持财政部作为最强大的角色（与历史相符），同时又不使设计变成一个单一策略的游戏。

有关完整版本演变过程，请参阅[`CHANGELOG.md`](./CHANGELOG.md)。

## 确定性

相同的种子+相同的玩家决策=在不同运行、浏览器和操作系统中，字节完全相同的记录。

- 单一随机数生成器：`mulberry32(state.rngSeed)`。
- 对手决策：纯函数，基于可见状态，每个决策都会记录在记录中，并附带触发该决策的规则。
- 保存/加载循环会保留状态哈希。
- 重播功能会从`initialState(seed) + decisionLog`重建。
- 在v0.2到v0.10的平衡弧中，经过1000多次确定性游戏的验证。

## 威胁模型和数据处理

《Sovereign》是一款独立的基于浏览器的棋盘游戏。CLI会在您的默认浏览器中打开一个本地HTML文件。没有服务器、网络调用、帐户或云同步。

- **访问的数据：**包括位于 `release/` 目录中的 HTML 文件（仅可读）以及位于 `sovereign.autosave` 键下的 `localStorage`（仅游戏存档状态）。
- **未访问的数据：**不访问包目录之外的任何文件系统，不进行任何类型的网络请求，不收集遥测数据，不进行分析，不使用任何凭据。
- **所需权限：**能够启动操作系统默认浏览器，能够读取包自身的文件，浏览器 `localStorage`（可选）。
- **绝不收集遥测数据。**模拟器的“遥测”功能是指从浏览器内账本中提取的本地游戏分析报告；这些报告绝不会离开您的设备。

有关漏洞报告和完整的安全策略，请参阅 [`SECURITY.md`](./SECURITY.md)。

## 特性

- **单人赛道胜利游戏**，对抗两个预设的对手（默认情况下为财政部/金融部和商人/基础设施部；制造商/工业部可用于批量游戏）。当一名玩家完成其第四次穿越财政部时，游戏结束。在最终结算时，影响力最高者获胜。平均回合数为 22 回合/66 步。
- **对手存在感（v1.5 版本）**——可见的影响力排名和每个对手相对于您在比赛中的地位的行动线（“汉密尔顿——影响力领先 3 点——收购银行；财政部联盟加强。”）。结束了平行单人游戏的体验；对手表现得像真正的对手。仅为展示，绝不会写入哈希账本。
- **信用螺旋（v1.5 版本）**——公共信用失败现在会产生影响，并可以进行修复。在信用较低时，会产生债务偿还现金费用，预示着加速走向违约，并预测趋势，改革行动则是一条真正的救命稻草。它直接传达了公民理念：您会感受到联邦公共信用的重要性。它涵盖了现有的失败等级（公众怀疑→危机→恐慌→违约），并且保存/重播功能完全是确定性的。
- **特效 + 音效（v1.5 版本）**——数字过渡，具有增益/损失不对称性，13 种提示音的 ZzFX 程序化音频，动作编排，以及速度设置（电影/正常/快速——快速模式跳过所有动画，以实现快速游戏和可访问性）。在整个游戏中都支持完整的键盘/减少动画/屏幕阅读器。
- **战略深度（v1.2 版本）**——三个配置文件锁定的特殊行动（发行联邦债券/协商路线合同/特许工坊），6 张具有时间窗口的手牌（手牌上限为 2），改革恢复行动。
- **战略弧线（v1.3 版本）**——从第 8 回合开始，每回合触发 8 个联邦时代事件（5 个选择 + 3 个自动），3 个配置文件愿景（联邦信用架构师/商业主权者/工业奠基人），并具有游戏结束奖励。所有三个愿景都是可以实现的。
- **编年史家（v1.4 版本）**——以第三人称叙述的历史声音。14 个与事件相关的横幅（《法案》× 7 / 联邦时代开始 / 怀疑 / 危机 / 恐慌 / 违约 / 叛乱 / 改革 / 愿景 / 最终结算）。所有引用的内容均已与 founders.archives.gov、Wikisource 和国会图书馆的资料进行核对。失败的《法案》被叙述为与真实历史相反的情况（“在我们的历史中，汉密尔顿的《融资法案》于 1790 年 7 月以 32 比 29 的票数通过；在您的共和国中，士兵的歧视获得了足够的选票，以阻止这一法案。”）。持久的带边框的提示，带有 × 按钮进行关闭；尊重叙述开启/最小/关闭设置。
- **编年史家 B 级——信息层（v1.5）**——15 个关于关键机制的“了解更多”弹出窗口，**编年史家的账本**百科全书（27 个经过验证的历史引文，以及《法案》、联邦时代事件、信用等级和愿景，全部在一个参考叠加层中），以及 10 个术语表提示。将时代风味转化为实际的、可浏览的历史层。
- **新手引导（v1.5 版本）**——一个 Swift-Start 引导的“1790 年融资辩论”开场，引导新手玩家进入核心循环，此外还有一个不隐藏的悬停/焦点提示，在您做出承诺之前，会显示每个行动的成本和后果。
- **确定性 AI**——每个对手的决策都是可见状态的纯函数，并带有记录在案的理由。没有 LLM，没有不透明的魔法。
- **8 个游戏界面**——主界面、财政部面板、资产检查器、事件抽屉、《国会法案》、共享轨道、回合日志/账本、游戏结束报告。
- **拍卖**——被拒绝的资产将进行多玩家拍卖，并根据配置文件进行预设竞标。
- **保存/加载**——每回合自动保存到 `localStorage`，手动 JSON 导出/导入，加载时进行哈希完整性检查，并根据版本进行限制。
- **重播**——可以对任何已完成的游戏进行完全的滚动。仅可读。从种子 + 决策日志重建，并带有绿色的完整性标记。
- **批量模拟**——针对任何配置文件三元组运行 10 / 50 / 100 个确定性游戏，并导出 JSON + HTML 报告以进行平衡分析。
- **历史叙述**——来自账本的 25 个条目的库（默认 40-60 字，扩展 150-200 字，游戏结束时约 300-500 字的共和国摘要）。绝不会改变状态。
- **可访问性基础**——完整的键盘导航、焦点指示器、屏幕阅读器可识别的标签、轨道值可以作为文本而不是标记进行读取，最小字体大小为 14 像素，尊重减少动画的设置。

## 配置文件（v0.10 平衡基准）

| 配置文件 | 资产优先级 | 优势 | 劣势 |
|--------------------------------|---------------------------------------------------------------|----------------------|-------------------------------------|
| **Treasury / Finance**         | NF > 国家债务 > 革命债务 > 银行 > 铸币厂 | 公共信用增长 | 没有基础设施收入 |
| **Merchant / Infrastructure**  | 路线（全部 4 条）> 商业 > 改进 > 收入 | 路线等级 | 没有工业产能评分 |
| **Manufacturer / Industry**    | 制造 > 战略工业 > 改进 > 银行 | 产能倍增器 | 起步缓慢；获得初始特许经营权 |

第四个概念文档配置文件（机会主义者/现金）已推迟。锁定的 v0.10 竞争集为三个。

## 测试版说明

- **这是一个公开的测试版本**——经过了完善，并且很有趣，但仍在不断改进；您可能会遇到一些小问题。欢迎在[问题跟踪器](https://github.com/mcp-tool-shop-org/sovereign/issues)上提交错误报告和反馈。
- **先完成棋盘并不意味着您会获胜。**游戏在有玩家完成第四圈时结束，但最终获胜者是拥有最多“影响力”的玩家——经济实力胜过速度。游戏结束界面会清楚地说明这一点。
- **“财政”是最强大的策略，这是设计的结果。**公共信贷和联邦财政是汉密尔顿的主要手段，因此“财政”策略通常会获胜——但“商人”和“制造商”策略都是切实可行的，并且玩法截然不同。
