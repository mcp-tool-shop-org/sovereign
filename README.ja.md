## 概要

Sovereignは、アメリカの公共信用制度の創設に関する**ハミルトン・システム・モノポリー形式のボードゲーム**であり、さらに**ソロ／デジタル版**として、ブラウザ上で同じルールをローカルで実行し、2つの決定論的なスクリプト化された対戦相手と対戦できるようにしたものです。

- **ボードゲーム** — 印刷可能な34ページの版。40マス、22の物件＋4つのルート＋2つの機関、8つのカラーシステム、固定された歴史的順序で並べられた7つの議会法、4つのプレイヤーロール、3つの共有トラック（公共信用、公共抵抗、工業能力）、12+12のイベントカード。財務省以外に、商人、製造業者という2つの実行可能な経済的経路。v0.2のバランスは固定。
- **デジタルモード** — 単一の独立したHTMLファイル。回路ベースの終了条件：いずれかのプレイヤーが財務省を4回通過するとゲームは終了します。平均ゲーム時間**約22ラウンド（約66ターン）**。最終会計では、最も影響力のあるプレイヤーが勝利します。必ずしも最初にボードを一周したプレイヤーではありません。ライバルの存在を示す要素：可視の影響力ランキング＋対戦相手ごとの姿勢を示す線。戦略的な深みを示す要素：3つのプロファイルにロックされた特別なアクション、タイミングウィンドウを持つ6つのハンドカード、改革による回復アクション、多段階のクレジット・スパイラル（公共の疑念→危機→パニック→デフォルト）と、負債の返済、デフォルトへの加速、予測、そして改革の道筋。戦略的な展開を示す要素：8ラウンドから各ラウンドで発生する8つの連邦時代のイベント、ゲーム終了時にボーナスを与える3つのプロファイルビジョン。クロニクラーによるナレーション：14個のイベントに関連付けられた歴史的なバナーと、有益なティアB（詳細はこちらポップオーバー、クロニクラーの記録百科事典、用語解説）。連邦主義者の論文と創設者オンラインからの実際の引用、×ボタンで閉じることができる永続的なメッセージ。視覚効果とサウンド：数値のアニメーション、ZzFXによる手続き型オーディオ（13種類のキュー）、映画のような／通常／高速のスピード設定。決定論的なmulberry32 RNG、スクリプト化されたAI対戦相手、ハッシュの整合性を持つ保存／読み込み、リプレイスクラバー、デザイナーが制御するバッチシミュレーションツール。
- **バランスの基準** — 回路＋戦略的な深み＋戦略的な展開＋クロニクラー＋クレジット・スパイラル（v1.5.0ベータ版）：財務省**48％**、商人**34％**、製造業者**18％**（CANONICAL × 100、`test/measure-stats.mjs`を使用してライブエンジンに対して測定）。3つのプロファイルすべてが意味のある勝利を収め、歴史的なテーマに沿って財務省が最も強力。3つのプロファイルビジョン（連邦信用アーキテクト／商業主権者／工業創設者）はすべて実現可能であり、おおよそバランスが取れています—それぞれ約41〜43％のゲームで発生します。基盤となるv0.18のメカニズム（クレジット危機、キャッシュによるIPスコアリング、工業憲章、セット完了ボーナス）は、1,000回以上の決定論的なシミュレーションゲームによって駆動されたv0.3→v0.10→v0.18のデザインサイクルから、バイト単位で完全に同一に保存されています。

## なぜこれがあるのか

Sovereignの基本的な考え方は、**公共信用と連邦政府の財政**が、アレクサンダー・ハミルトンの経済政策における主要な要素であったということです。しかし、ハミルトンのシステムをゲームとして実装する場合、**商業**と**産業**もまた、勝利への有効な道である必要があります。バランス調整（v0.2 → v0.10）は、9つのバージョンにわたる、証拠に基づいた調整であり、財務省を最も強力な要素として維持し（歴史的事実と整合性を持たせる）、ゲームを単一の戦略に限定しないようにすることを目的としています。

各バージョンの詳細な変更点は、[`CHANGELOG.md`](./CHANGELOG.md)を参照してください。

## 決定論

同じシード値と、同じプレイヤーの決定を行うと、実行、ブラウザ、オペレーティングシステムを問わず、完全に同一のゲームデータが生成されます。

- 単一の乱数生成器：`mulberry32(state.rngSeed)`.
- 相手の決定：可視状態のみを引数とする純粋関数であり、すべての決定は、その決定を引き起こしたルールとともにゲームデータに記録されます。
- 保存/読み込みのサイクルでは、状態のハッシュ値が保持されます。
- リプレイ機能では、`initialState(seed) + decisionLog`からゲームを再構築します。
- v0.2 → v0.10のバランス調整中に、1,000以上の決定論的なゲームで検証されました。

## 脅威モデルとデータ処理

Sovereignは、スタンドアロンのブラウザベースのボードゲームです。CLI（コマンドラインインターフェース）は、デフォルトのブラウザでローカルのHTMLファイルを開きます。サーバー、ネットワーク接続、アカウント、クラウド同期はありません。

- **アクセスされるデータ：** `release/`ディレクトリに含まれるHTMLファイル（読み取り専用）と、`sovereign.autosave`キーの下の`localStorage`（ゲームの保存状態のみ）。
- **アクセスされないデータ：** パッケージディレクトリ外のファイルシステムへのアクセス、あらゆる種類のネットワークリクエスト、テレメトリー、分析、認証情報はありません。
- **必要な権限：** OSのデフォルトブラウザを起動する能力、パッケージ自体のファイルを読み取る能力、ブラウザの`localStorage`（オプション）。
- **テレメトリーは一切行いません。** シミュレーターの「テレメトリー」機能とは、ブラウザ内で生成されるゲームデータから派生した、ローカルのゲーム分析レポートを指します。これらのレポートは、決してあなたのコンピューターから送信されません。

脆弱性報告と完全なセキュリティポリシーについては、[`SECURITY.md`](./SECURITY.md)を参照してください。

## 機能

- **ソロでの勝利を目指すゲーム**。対戦相手は、事前に設定された2つのプロファイル（デフォルトでは「財務省/金融」と「商業/インフラストラクチャ」、バッチプレイでは「製造/産業」も利用可能）。ゲームは、いずれかのプレイヤーが「財務省」への訪問を4回完了した時点で終了。平均ラウンド数は約22ラウンド、ターン数は約66ターン。最終会計時の影響力が最も大きいプレイヤーが勝利。
- **ライバルの存在（v1.5レイヤー）**：各ライバルの影響力ランキングと、レースにおける自身の立ち位置を基準とした各ライバルの戦略を示す線が表示される（例：「ハミルトンは影響力で3ポイントリードしており、銀行を狙っている。財務省の陣営は結束を強めている」）。これにより、単独プレイのような感覚がなくなり、対戦相手が明確な存在として認識される。これはあくまで表示上の変更であり、ゲームデータには影響しない。
- **クレジット・スパイラル（v1.5レイヤー）**：公共クレジットの失敗が、ゲームに影響を与えるようになり、その影響は累積し、回復も可能になった。クレジットが低い場合、債務返済のための現金が徴収され、デフォルト（債務不履行）への進行が早まり、その結果が予測可能になる。また、改革アクションが、現実的な解決策として提示される。これにより、連邦政府の公共クレジットがなぜ重要だったのかが、ゲームを通して実感できる。v0.18の階層構造（公共の疑念→危機→パニック→デフォルト）は変更されないが、`reduce()`関数内で適用されるため、リプレイの安全性は維持される。
- **効果音と視覚効果（v1.5レイヤー）**：数値の変化に合わせたアニメーション、13種類の効果音、アクションの演出、およびゲーム速度の設定（映画のような演出/通常/高速 - 高速モードではアニメーションをすべてスキップし、ゲームプレイを高速化し、アクセシビリティを向上）。キーボード操作、モーションの少ない表示、スクリーンリーダーへの対応。
- **戦略の深み（v1.2レイヤー）**：3つのプロファイルにロックされた特別なアクション（連邦債の発行/ルート契約の締結/ワークショップの設立）、タイミングが重要な6枚の「ハンド」カード（ハンド上限は2枚）、改革による回復アクション。
- **戦略の展開（v1.3レイヤー）**：8ラウンド以降、各ラウンドで5つの選択肢と3つの自動イベントからなる、8つの連邦時代イベントが発生。3つのプロファイルビジョン（連邦クレジットアーキテクト/商業主権者/産業創始者）があり、ゲーム終了時に+3のIP（影響力ポイント）ボーナスが付与される。3つのビジョンはすべて達成可能（それぞれゲームの約41〜43％、CANONICAL × 100）。
- **記録者（v1.4レイヤー）**：三人称視点の歴史的な語り口。14種類のイベントに関連するバナー（7つの法案/連邦時代の開始/疑念/危機/パニック/デフォルト/反乱/改革/ビジョン/最終会計）。すべての引用は、founders.archives.gov、Wikisource、および米国議会図書館の資料と照合して検証されている。失敗した法案は、現実の歴史とは異なる状況として語られる（例：「私たちの歴史では、ハミルトンの資金調達法は1790年7月に32対29で可決された。しかし、あなたの共和国では、兵士に対する差別が十分な票を集め、その法案を阻止した」）。ゲーム中に表示される、フォイル加工されたメッセージ。表示設定（オン/最小/オフ）を尊重する。
- **記録者ティアB - 情報レイヤー（v1.5）**：主要なゲームメカニズムに関する15個の「詳細はこちら」ポップアップ、**記録者の記録**百科事典（27個の検証済みの歴史的引用、および法案、連邦時代イベント、クレジット階層、およびビジョンをまとめたもの）、および10個の用語集ツールチップ。これにより、ゲームの時代背景が、実際に参照できる歴史レイヤーとして表現される。
- **チュートリアル（v1.5レイヤー）**：Swift-Startガイド付きの「1790年資金調達に関する議論」というオープニングにより、初めてプレイするプレイヤーがゲームの基本的な流れを理解できるようになる。また、すべての操作とその結果が、操作前に表示されるため、何も隠すことなくゲームを体験できる。
- **決定的なAI**：対戦相手のすべての決定は、表示されているゲームの状態と記録された理由に基づいて決定される純粋な関数である。LLM（大規模言語モデル）や不透明な要素は使用されていない。
- **8つのゲーム画面**：ゲームボード、財務省パネル、アセットインスペクター、イベントドロワー、議会の法案、共有トラック、ターンログ/記録、ゲーム終了レポート。
- **オークション**：売却されなかったアセットは、プロファイルに基づいて設定されたスクリプトに従って入札が行われるマルチプレイヤーオークションにかけられる。
- **セーブ/ロード**：すべてのターンで`localStorage`に自動保存、手動でのJSONエクスポート/インポート、ロード時のハッシュ整合性チェック、バージョンによる制限。
- **リプレイ**：完了したゲームの任意の時点を再生できる。読み取り専用。シードと決定ログから再構築され、整合性が確認される。
- **バッチシミュレーション**：任意の3つのプロファイルに対して、10/50/100回の決定的なゲームを実行し、バランス分析のためにJSON + HTMLレポートをエクスポートする。
- **歴史的なナレーション**：記録から派生した25個のエントリーのライブラリ（デフォルトは40〜60語、拡張は150〜200語、ゲーム終了時の共和国の概要は約300〜500語）。ゲームの状態は変更されない。
- **アクセシビリティ**：キーボード操作、フォーカスインジケーター、スクリーンリーダーに対応したラベル、トラックの値がテキストとしても表示される、最小フォントサイズ14px、モーションの少ない表示。

## プロファイル構成（v0.10バランス基準）

| プロファイル | アセットの優先順位 | 強み | 弱み |
|--------------------------------|---------------------------------------------------------------|----------------------|-------------------------------------|
| **Treasury / Finance**         | NF > 州債 > 革命債 > 銀行 > 造幣局 | 公共クレジットの増加 | インフラ収入なし |
| **Merchant / Infrastructure**  | ルート（すべて4つ）> 商業 > 改善 > 収入 | ルートの階層 | 産業能力によるスコアリングなし |
| **Manufacturer / Industry**    | 製造 > 戦略的産業 > 改善 > 銀行 | 能力の倍増効果 | 序盤は苦戦するが、開始時にワークショップを設立できる |

4番目のコンセプトドキュメントのプロファイル（機会主義者/現金）は、今後のバージョンで実装される。v0.10のロックされた競争セットは3つである。

## 既知の注意点

- **v1.5.0は、実際の人間による完全なプレイテストを待つベータ版です。** 各レイヤーは構造的に監査され、`test/measure-stats.mjs`および決定性＋プレイアビリティのテストを通じて、ライブエンジンに対して再検証されました。完全なシステムはスライスレベルでテストされましたが、まだ実際の人間プレイヤーによる完全なプレイテストは行われていません。そのプレイテストは、パブリック版のリリースにおけるプレイアビリティの基準となります。以下に示す数値は、ライブエンジンに対する正規値の100倍です。実際の人間によるプレイでは、数値が異なる場合があります。あなたが（または信頼できる人物）がプレイテストを行うまで、このバージョンはオプションとして扱ってください。
- **プレッシャーは感じられ、回復可能です。単なる装飾ではありません。** 「Credit Crisis」は、現在、約29/100のゲームで発生し、実際に回復可能です。危機状態（Credit ≤ 4）に陥ったゲームのうち、約41%が安定した状態（Credit ≥ 7）に戻り、デフォルト状態になるゲームはありません。パニックはまれです（約1/100）。デフォルトと反乱は、スクリプト化されたv0.18のAIの下ではまれですが（AIは、崩壊する前に自らを修正します）、パブリックCreditを無視する人間プレイヤーであれば、どちらも完全に達成可能です。Credit Spiralにより、デフォルトへの傾斜が、急な崖ではなく、より明確に感じられるようになります。
- **対戦相手はv1.2～v1.4のシステムを使用します。コアとなるアクションの計算のみがv0.18です。** スクリプト化された対戦相手は、特殊アクション、改革の手段、連邦時代／終盤の選択、投票、HANDカードのタイミングを使用します。以前の「AIは適応しない」という注記は、範囲が広すぎました。v0.18のまま残っているのは、**コアとなる購入／オークション／アップグレード／投票**の評価です。対戦相手は、それぞれのプロファイルに基づいて最適な選択をしますが、まだ人間が最適化する場合のように、明示的に「Visionを目指す」わけではありません。正規値の100倍で測定された数値は、スクリプト化された行動を反映しています。人間によるプレイでは、数値が異なる場合があります。
- **トリガー＝勝者ではありません。** 4回目のサーキットを完了したプレイヤーが、約3分の1のゲームで影響力によって勝利します。これは意図的なものであり、最終的な会計は、ボード上でのスピードではなく、経済的な深さを評価します。終盤のコピーでは、この点が明確に説明されています。
- **連邦時代は、比較的軽いActのプレッシャーを持ちます。** 創設Actは、1～7ラウンドで発生します。平均的なプレイは、約22ラウンドです。したがって、連邦時代は、独自のイベント（8ラウンドから毎ラウンド発生）とCredit Spiral、Visionレースによって進行します。v1.3では、毎ラウンド連邦時代のカードを引くことで、4ラウンドの空き期間が約2/100に減少しました。それでも、プレイテストで期間が短いと感じられる場合は、次の対策として、Actの再配分を行い、以前の状態に戻すことはありません。
- **財務／金融は、意図的に最も強力な状態を維持します**（勝利の48%）。これは、歴史的な仮説と一致します。パブリックCreditと連邦金融は、ハミルトンの主要な経済的手段でした。ただし、デザイン全体を単一の戦略（商人34%、製造業者18%）に押しやることはありません。

## ビルドと貢献

```bash
git clone https://github.com/mcp-tool-shop-org/sovereign
cd sovereign
npm install
npm test           # smoke tests
npm run verify     # full verify (smoke + pack-dry-run + CLI flag check)
npm run play       # open the game locally
```

リリースは、GitHub Actions（`release.yml`）を通じて、`v*`タグのプッシュ時にnpmに公開され、Sigstoreによるプロビナンス認証が行われます。ソースコードの信頼できる場所は、`main`ブランチです。
