## 概要

Sovereignは、**アメリカの公的信用に関するハミルトン・システムを基にしたボードゲーム**であり、さらに、**完全なソロ/デジタル版**も付属しています。このデジタル版は、同じルールに基づいて、ブラウザ上で2つの決定的なAI対戦相手とローカルで動作します。

- **ボードゲーム** — 34枚のシートで印刷可能なバージョン。40マス、22の物件 + 4つのルート + 2つの機関、8つのカラーシステム、固定された歴史的な順番で配置された7つの議会法、4つのプレイヤーロール、3つの共通トラック（公共信用、公共の抵抗、工業生産能力）、12+12枚のイベントカード。財務省以外にも、商人、製造業者という2つの有効な経済的戦略が存在します。v0.2のバランス調整は完了しており、変更はありません。
- **デジタルモード** — 単一の自己完結型のHTMLファイル。circuitに基づいたゲーム終了条件：プレイヤーが「財務省開放」を4回達成するとゲームが終了します。平均ゲーム時間は約23ラウンド（67ターン）。最終決算では、影響力（Influence）が最も高いプレイヤーが勝利します。必ずしも最初にボードを一周したプレイヤーが勝利するとは限りません。戦略的な深み：プロファイルに紐づいた特殊アクション3つ、タイミングが設定されたHANDカード6枚、改革による回復アクション、段階的な信用への圧力（公共の疑念、危機、パニック、デフォルト）。戦略的な展開：連邦時代イベントが8ラウンド目以降、毎ラウンドで発生、エンディング時のボーナスが付与されるプロファイルビジョン3つ。年代記：14個のイベントに関連する歴史的なナレーション、連邦主義論文やFounders Onlineからの実際の引用文、×ボタンで消去できるポップアップ表示。決定論的な mulberry32 RNG、スクリプトによるAI対戦相手、ハッシュ整合性によるセーブ/ロード機能、リプレイの削除機能、開発者限定のバッチシミュレーションツール。
- **バランスの基準** — circuit + 戦略的な深み + 戦略的な展開 + 年代記（v1.4.0ベータ版）：財務省 59%、商人 20%、製造業者 21%（標準値 × 100）。3つのプロファイルすべてが意味のある勝利を得られるように設計されています。ビジョン達成率：連邦信用設計者 54%、商業主権者 39%、工業創業者 29%。基盤となるv0.18のメカニズム（信用危機、現金IPスコアリング、工業特許、セット完了ボーナス）は、1,000以上の決定論的なシミュレーションゲームによって駆動されたv0.3 → v0.10 → v0.18のデザインアーキテクチャから、完全に同一の状態で引き継がれています。

## 始め方

### ブラウザでプレイ (インストール不要)

```bash
npx @mcptoolshop/sovereign
```

CLI（コマンドラインインターフェース）を使用すると、ゲームがデフォルトのブラウザで開きます。インストーラー、サーバー、インターネット接続は不要です。

その他のモード：

```bash
npx @mcptoolshop/sovereign --print    # Open the printable 34-sheet board game
npx @mcptoolshop/sovereign --start    # Open the audience-routed landing page
npx @mcptoolshop/sovereign --path     # Print the playable HTML file path
npx @mcptoolshop/sovereign --help     # All flags
```

### オンラインでプレイ

**<https://mcp-tool-shop-org.github.io/sovereign/>** にあるランディングページを開き、デジタルゲームをクリックします。

### 印刷してプレイ

ボードゲームのプロトタイプは、自己完結型の34ページのHTMLドキュメントです。パッケージ内の `release/board-game/sovereign-prototype.html` を開くか、ダウンロードしたファイルを開き、`Cmd/Ctrl-P → PDFとして保存 → US Letter → 100%スケール` で保存します。カットしてプレイしてください。

### オフラインリリースバンドル

各リリースには、GitHubのリリースページに `sovereign-vX.Y.Z-release.zip` というバンドルが添付されています。ダウンロードして解凍し、`00-START-HERE.html` を開いてください。すべてオフラインで動作します。

## このゲームが存在する理由

Sovereignのテーマは、**公的信用と連邦金融**がアレクサンダー・ハミルトンの経済政策における主要な要素であったというものです。しかし、ハミルトン・システム・ゲームでは、**商業**と**産業**も勝利への有効な道であるべきです。v0.2からv0.10までのバージョンアップは、財務省を最も強力な戦略として維持しつつ（歴史的な事実に基づき）、ゲームが単一の戦略に偏らないようにするための、9つのバージョンにわたる、証拠に基づいた調整でした。

詳細なバージョンごとの変更履歴は、[`CHANGELOG.md`](./CHANGELOG.md) を参照してください。

## 決定性

同じシード値と、同じプレイヤーの行動 = 実行ごとに、ブラウザやオペレーティングシステムに関わらず、完全に同一の記録。

- 単一の乱数生成器：`mulberry32(state.rngSeed)`。
- 相手の行動：可視状態のみに基づいて決定される純粋な関数。すべての行動は、トリガーとなるルールとともに記録に記録されます。
- セーブ/ロード：状態のハッシュ値を保持します。
- リプレイ：`initialState(seed) + decisionLog` から再構築されます。
- v0.2からv0.10までのバランス調整における1,000回以上の決定的なゲームで検証されています。

## 脅威モデルとデータ取り扱い

Sovereignは、スタンドアロンで動作するブラウザベースのボードゲームです。CLI（コマンドラインインターフェース）は、デフォルトのブラウザでローカルのHTMLファイルを起動します。サーバー、ネットワーク接続、アカウント、クラウド同期は一切ありません。

- **アクセスされるデータ:** `release/`ディレクトリに含まれるHTMLファイル（読み取り専用）、および`sovereign.autosave`キーの下にある`localStorage`（ゲームのセーブデータのみ）。
- **アクセスされないデータ:** パッケージディレクトリ外のファイルシステムへのアクセス、あらゆる種類のネットワークリクエスト、テレメトリー、分析、認証情報。
- **必要な権限:** OSのデフォルトブラウザを起動する能力、パッケージ自身のファイルを読み取る能力、ブラウザの`localStorage`（オプション）。
- **テレメトリーは一切行いません。** シミュレーターの「テレメトリー」機能は、ブラウザ内の台帳から派生したローカルのゲーム分析レポートを指します。これらはあなたのマシンから一切外部に出ません。

脆弱性報告およびセキュリティポリシーの詳細については、[`SECURITY.md`](./SECURITY.md) を参照してください。

## 機能

- **ソロプレイ：勝利条件は、対戦相手（デフォルトでは財務/金融と商業/インフラストラクチャ、バッチプレイでは製造/産業を選択可能）を2人相手に勝利すること。ゲームは、プレイヤーが「財務開示」を4回完了した時点で終了。平均23ラウンド/67ターン。最終的な会計において、最も高い影響力を持つプレイヤーが勝利。
- **戦略的深み（バージョン1.2）：** プロファイルに紐づいた3つの特殊アクション（連邦債の発行、仲介契約、ワークショップの設立）、タイミングを考慮した6枚の手札（手札上限2枚）、改革による回復アクション、段階的な信用リスク（国民の疑念、危機、パニック、デフォルト）。
- **戦略的展開（バージョン1.3）：** ラウンド8以降、毎ラウンドに8つの連邦時代イベントが発生（5つの選択肢と3つの自動イベント）。3つのプロファイルビジョン（連邦信用アーキテクト、商業主権、産業創業者）があり、ゲーム終盤に+3のIPボーナスが付与される。
- **年代記（バージョン1.4）：** 歴史的な視点から語られる第三者視点によるナレーション。14種類のイベントに関連するバナー（法案×7、連邦時代の幕開け、疑念、危機、パニック、デフォルト、反乱、改革、ビジョン、最終会計）。引用文は、founders.archives.gov、Wikisource、および米国議会図書館の資料に基づいて検証済み。失敗した法案は、歴史的事実とは異なる「もし～ならば」という形で語られる（例：「我々の歴史では、ハミルトンの資金法は1790年7月に32対29で可決されました。しかし、あなたの共和国では、兵士による差別が十分な票を集め、扉を閉ざしました。」）。フォイル加工された装飾は、表示/非表示のオプションがあり、ナレーションの音量設定（オン/最小/オフ）に対応。
- **決定論的なAI：** 対戦相手のすべての意思決定は、可視化された状態に基づいており、その理由が記録されている。大規模言語モデルや不透明な魔法は使用されていない。
- **8種類のゲーム画面：** ボード、財務パネル、資産検査画面、イベント引き出し、議会法案、共有トラック、ターンログ/記録、ゲーム終盤レポート。
- **オークション：** 獲得を拒否された資産は、プロファイルに基づいたスクリプトによる入札が行われるマルチプレイヤーオークションにかけられる。
- **セーブ/ロード：** ターンごとに`localStorage`に自動保存。手動でJSON形式でエクスポート/インポート可能。ロード時にハッシュ整合性チェックを実施。バージョンによって機能が制限される場合がある。
- **リプレイ：** 完了したゲームを完全に再現可能。読み取り専用。シードデータと意思決定ログから、整合性を示す緑色のインジケーターと共に再構築される。
- **バッチシミュレーション：** 任意のプロファイル組み合わせに対して、10/50/100回の決定論的なゲームを実行し、JSON形式とHTML形式のレポートをエクスポートして、バランス分析を行う。
- **歴史的ナレーション：** 記録データから派生した25種類のナレーション（デフォルトでは40～60語、拡張版では150～200語、ゲーム終盤の共和国に関する要約は300～500語）。ゲームの状態を変更することはない。
- **アクセシビリティ：** キーボードによる完全なナビゲーション、フォーカスインジケーター、スクリーンリーダーに対応したラベル、数値のテキスト表示（マーカーだけでなく）、最小フォントサイズ14px、アニメーションの速度調整に対応。

| プロファイル | 資産優先順位 | 強み | 弱み |

## 既知の制限事項

- **v1.4.0はベータ版です。** 各機能（戦略深度、戦略的展開、記録者）について、構造的な検証を行い、シミュレーションの結果と照合しました。4層構造全体は、各機能レベルで検証は行われましたが、まだ人間が実際にプレイしたわけではありません。数値データは、CANONICAL × 100の基準に基づいています。100回プレイした場合、約100回でゲーム終了、中央値は23ラウンド、勝利者の内訳は59勝/20勝/21勝、1ゲームあたり約6～8個の記録者バナーが出現、約33個の反応が発生します。実際にプレイするまでは、あくまで試用版として扱ってください。
- **AIの動作は、v1.2～v1.4のゲームシステムにまだ対応していません。** v0.18の意思決定ロジックを使用しており、人間プレイヤーのように「ビジョンを競い合う」ことや「戦略的にハンドカードを使用する」ことはありません。実際の人間によるプレイは、CANONICAL × 100の数値データとは異なる結果になる可能性があります。
- **トリガー＝勝利者、ではありません。** 4つ目の回路を完了したプレイヤーが勝利するのは、ゲームの約3分の1の場合のみです。これは意図的なものであり、最終的な勝利は、ボード上でのスピードだけでなく、経済的な深さによって決まります。ゲーム終盤の描写で、この違いが明確になります。
- **共和政後期（Late Republic）の期間は長く、イベント（Acts）は発生しません。** イベントは、1～7ラウンドでのみ発生します。平均的なプレイ時間は約23ラウンドで、そのうち約16ラウンドは新しい政治的なイベントが発生しない共和政後期となります。もし、この期間が単調に感じられる場合は、次のアップデートでイベントの再配置を行う予定です。強制的なイベントの追加ではありません。
- **財務/金融機能は、意図的に最も強力なものとして設計されています。** これは、歴史的な背景に基づいています。ハミルトンにとって、公的信用と連邦政府の財政は、最も重要な経済的手段でした。
- **失敗イベント（デフォルト/反乱）は、ほとんど装飾的なものです。** 信用危機は時々発生しますが、デフォルトや反乱はほとんど発生しません。エスカレーションシステムは、より多くの時間で影響を及ぼしますが、それでもほとんどの場合、上位レベルには到達しません。今後のバージョンでは、失敗状態へのプレッシャーについて再検討する可能性があります。
