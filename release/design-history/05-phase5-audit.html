Phase 5 Audit

Narration · Replay · Save / Load — additive-layer review.

Phase 5 was scoped as a purely additive layer on top of the frozen Phase 4 scripted-opponents baseline: narration library, replay UI, save / load, and an autosave. This audit verifies that the layer mutates no rule, no reducer, no balance value, and no opponent policy; that save and load round-trip with hash integrity; that the replay surface is read-only and deterministic; and that the nine acceptance criteria from the Phase 5 brief are met. Source under audit: sovereign-solo-phase5-narration-replay-save.html. Baseline: sovereign-solo-phase4-scripted-opponents.html.

§ 01Method

The audit ran static byte-diffing first, runtime probing second, and a manual playthrough only where the prior two were insufficient. The Phase 4 baseline and Phase 5 candidate were compared at the level of the named code blocks the brief calls out as load-bearing: the reducer, the seventeen action shapes, the opponent profile dispatch, the act table, the asset table, the route ladder, the space table, both card decks, scoring, rent computation, and initialState(). Where the Phase 5 file overrode or wrapped a Phase 4 function, the wrapper was inspected line-for-line for any path that could mutate game state.

Runtime probing exercised the four scenarios where save / load behavior is most likely to drift: a clean save → load round-trip on a partial game, four malformed payloads (bad version, missing seed, malformed log, null payload), a tampered finalState.hash, and a same-seed determinism check on two independently restarted games. The Phase 4 voting orchestration debt — already in the baseline — was reproduced and classified, not patched.

§ 03Narration Layer

The narration library is a frozen data object keyed by trigger. Each entry has three fields: a heading, a default snippet, and an expand snippet (the endgame republic-summary is the exception — it carries empty default / expand strings and is rendered through buildRepublicSummary(state) at game over). Twenty-five trigger keys are defined: eight first-purchase, seven act-passed, six track-threshold, three endgame-winner, and one endgame-summary.

§ 08Determinism

Two clean restarts on seed 2026 with one human CAST_VOTE dispatched produced identical state hashes and identical ledger lengths. The save → load round-trip on a 17-action partial game produced an identical hash to the live state. mulberry32(state.rngSeed) remains the only RNG source; no Math.random or Date.now call influences game truth. The dispatch wrapper introduces setTimeout only for UI pacing of opponent moves — the underlying state transition is synchronous through reduce(), and replay omits the timer entirely.

§ 10Polish Discipline

  • No runtime placeholder pills or demo banners in the live UI. The only placeholder occurrence is a legitimate HTML attribute on the auction bid input (placeholder="${minBid}").
  • No developer-aside comments. Grep for “TODO”, “FIXME”, “XXX”, “hmm”, “let me” returned zero hits in narrative source.
  • No external assets, fonts, or network dependencies. All typography is system-stack; no cdn, googleapis, or absolute URL anywhere in the file.

§ 11Phase 4 Voting Caveat — Classification

The Phase 5 builder flagged a pre-existing Phase 4 orchestration issue: opponent Act votes are only dispatched when an opponent becomes the active player after a roll. Static inspection confirms this: dispatch() kicks runOpponent() only when STATE.players[STATE.activePlayerIndex].profile !== 'human' and the phase is one of awaiting-roll or act-vote. At lap start the active player is the human, so the trigger never fires from voting alone; in interactive play the human votes, the reducer does not finalize because the opponent slots remain null, and the loop stalls.

Classification: Phase 4 orchestration debt — inherited, pre-Phase-5, must not be patched at this freeze.

Risk to Phase 5 deliverables: none. Replay reconstructs from initialState(seed) + decisionLog, and the decision log is a faithful transcript of whatever did dispatch. If the caveat stalls a live game at an act vote, no END_TURN fires, no autosave is written, and the partial log replays cleanly to the stall point.

Risk to determinism: none. The caveat is in the event-loop scheduler, not in the reducer or the RNG.

Risk to integrity hashing: none. snapshotHash() reads the state the reducer produced; whatever state was produced will replay byte-identically.

The fix is a Phase 4 orchestration patch — schedule the opponent vote sweep on entry to act-vote rather than on activePlayerIndex rotation — and explicitly belongs to a future Phase 4 maintenance pass, not to Phase 5.

§ 12Non-Blocking Deviations

Deviation 1 — Opponent decisions stored in decisionLog. The brief prefers opponent decisions be omitted from the log on the grounds that they are deterministic from state. Phase 5 stores them. The trade is intentional: storing makes the log self-contained against any future change to opponent timing, profile data, or decision-function refactor, and the replay path can ignore the profile decision tree entirely. Save-file size grows linearly but remains small (a finished game well under 200 KB). Severity: Low — non-blocking; preserves replay integrity even if Phase 6 changes opponent behavior.

Deviation 2 — Decision-log metadata recomputed on load. loadFromPayload() reconstructs DECISION_LOG from the payload but overwrites playerIdx, turn, and lap fields with the values derived from the current replay state rather than trusting the saved metadata. The action sequence is what matters for state reconstruction, so this is harmless. Severity: Low.

§ 13Freeze Verdict

Pass

Phase 5 is freeze-ready. The reducer, action set, opponent profiles, scoring, rent computation, act table, asset table, route ladder, card decks, and initialState are all byte-identical to the Phase 4 baseline. Narration is pure data; the detector reads state, never writes it. Save / load round-trips with verified hash integrity, rejects four malformed payload shapes and one tampered finalState cleanly, and writes an autosave only at safe boundaries. The replay UI reconstructs frames from initialState(seed) + decisionLog through a pure reducer and surfaces an integrity indicator. Determinism through mulberry32(state.rngSeed) is preserved.

The two non-blocking deviations and the inherited Phase 4 voting caveat are documented above; none compromises save, load, replay, or integrity in the present file. No required fixes before freeze.
