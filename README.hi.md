## यह क्या है

"सोवरिन" एक **हैमिल्टन-सिस्टम मोनोपोली-शैली का बोर्ड गेम** है जो अमेरिकी सार्वजनिक ऋण की स्थापना के बारे में है, साथ ही एक **पूर्ण एकल/डिजिटल संस्करण** भी है जो समान नियमों को दो पूर्वनिर्धारित स्क्रिप्टेड विरोधियों के खिलाफ ब्राउज़र में स्थानीय रूप से चलाता है।

- **बोर्ड गेम** — मुद्रित करने योग्य 34 पृष्ठों का संस्करण। 40 स्थानों वाला बोर्ड, 22 संपत्ति + 4 मार्ग + 2 संस्थान, 8 रंग प्रणाली, 7 अमेरिकी कांग्रेस के अधिनियम जो एक निश्चित ऐतिहासिक क्रम में हैं, 4 खिलाड़ी भूमिकाएं, 3 साझा ट्रैक (सार्वजनिक ऋण · सार्वजनिक प्रतिरोध · औद्योगिक क्षमता), 12+12 घटना कार्ड। खजाने मंत्रालय के अलावा दो व्यवहार्य आर्थिक रास्ते हैं: व्यापारी और निर्माता। v0.2 का संतुलन, जो अपरिवर्तित है।
- **डिजिटल मोड** — एक एकल, आत्मनिर्भर HTML फ़ाइल। 12 राउंड का गेम जिसमें अनिवार्य जीत मॉडल है: 8वें राउंड से, 15 प्रभाव और 5 अंकों की बढ़त वाला खिलाड़ी "अंतिम लेखा" को सक्रिय करता है और गेम समाप्त हो जाता है। यदि कोई अनिवार्य जीत नहीं है, तो गेम 12वें राउंड में समाप्त हो जाता है। इसमें mulberry32 RNG (यादृच्छिक संख्या जनरेटर), स्क्रिप्टेड AI (कृत्रिम बुद्धिमत्ता) विरोधी (खजाने मंत्रालय / वित्त, व्यापारी / बुनियादी ढांचा, निर्माता / उद्योग), हैश अखंडता के साथ सहेजें / लोड करें, रीप्ले क्लीनर, और डिजाइनर-नियंत्रित बैच सिमुलेशन टूल शामिल हैं।
- **संतुलन आधार** — 12 राउंड का अनिवार्य मॉडल (v1.1.1 बीटा): खजाने मंत्रालय 51% · व्यापारी 33% · निर्माता 16% (मानक × 100)। तीनों प्रोफाइल अनिवार्य जीत प्राप्त कर सकते हैं; कोई भी प्रोफाइल प्रतिबंधित नहीं है। v0.18 की अंतर्निहित कार्यप्रणाली (क्रेडिट संकट, नकद IP स्कोरिंग, औद्योगिक चार्टर, सेट पूर्णता बोनस) को 1,000 से अधिक नियतात्मक सिमुलेशन गेम द्वारा संचालित v0.3 → v0.10 → v0.18 डिज़ाइन चक्र से बिल्कुल समान रखा गया है।

## शुरुआत कैसे करें

### अपने ब्राउज़र में खेलें (कोई इंस्टॉलेशन नहीं)

```bash
npx @mcptoolshop/sovereign
```

CLI आपके डिफ़ॉल्ट ब्राउज़र में गेम खोलता है। कोई इंस्टॉलर नहीं, कोई सर्वर नहीं, इंटरनेट की आवश्यकता नहीं है।

अन्य मोड:

```bash
npx @mcptoolshop/sovereign --print    # Open the printable 34-sheet board game
npx @mcptoolshop/sovereign --start    # Open the audience-routed landing page
npx @mcptoolshop/sovereign --path     # Print the playable HTML file path
npx @mcptoolshop/sovereign --help     # All flags
```

### ऑनलाइन खेलें

होस्ट किए गए लैंडिंग पृष्ठ पर जाएं **<https://mcp-tool-shop-org.github.io/sovereign/>** और डिजिटल गेम पर क्लिक करें।

### प्रिंट करें और खेलें

मुद्रण योग्य बोर्ड गेम एक आत्मनिर्भर 34-पृष्ठ HTML दस्तावेज़ है। पैकेज से `release/board-game/sovereign-board-game.html` खोलें (या किसी डाउनलोड से), फिर `Cmd/Ctrl-P → Save as PDF → US Letter → 100% scale`। काटें और खेलें।

### ऑफ़लाइन रिलीज़ बंडल

प्रत्येक टैग किए गए रिलीज़ में `sovereign-vX.Y.Z-release.zip` बंडल अपने GitHub रिलीज़ पृष्ठ से जुड़ा होता है। इसे डाउनलोड करें, अनज़िप करें और दर्शकों के लिए प्रवेश बिंदु के रूप में `00-START-HERE.html` खोलें। सब कुछ ऑफ़लाइन चलता है।

## यह क्यों मौजूद है

"सोवरिन" का सिद्धांत यह है कि **सार्वजनिक ऋण + संघीय वित्त** अलेक्जेंडर हैमिल्टन के लिए एक प्रमुख आर्थिक साधन थे - लेकिन एक हैमिल्टन-सिस्टम गेम को **व्यापार** और **उद्योग** को भी जीत के व्यवहार्य रास्ते बनाने की अनुमति देनी चाहिए। संतुलन चक्र (v0.2 → v0.10) नौ संस्करणों का, साक्ष्य-आधारित प्रयास था, जिसका उद्देश्य ट्रेजरी को सबसे मजबूत प्रोफाइल के रूप में बनाए रखना था (इतिहास के अनुरूप) बिना डिज़ाइन को एक-रणनीति वाले गेम में बदलने के।

पूर्ण संस्करण-दर-संस्करण विकास के लिए [`CHANGELOG.md`](./CHANGELOG.md) देखें।

## नियतिवाद

समान बीज + समान मानव निर्णय = रन, ब्राउज़र और ऑपरेटिंग सिस्टम में बाइट-समान खाता।

- एकल RNG: `mulberry32(state.rngSeed)`.
- विरोधी निर्णय: दृश्य स्थिति के शुद्ध फ़ंक्शन, प्रत्येक निर्णय को ट्रिगरिंग नियम के साथ खाता में लॉग किया जाता है।
- सहेजें/लोड राउंडट्रिप एक राज्य हैश को संरक्षित करता है।
- रीप्ले `initialState(seed) + decisionLog` से पुनर्निर्माण करता है।
- v0.2 → v0.10 संतुलन चक्र के दौरान 1,000 से अधिक नियतात्मक गेम में सत्यापित।

## खतरे का विश्लेषण और डेटा प्रबंधन

"सोवरिन" एक स्टैंडअलोन ब्राउज़र-आधारित बोर्ड गेम है। CLI आपके डिफ़ॉल्ट ब्राउज़र में एक स्थानीय HTML फ़ाइल खोलता है। इसमें कोई सर्वर नहीं है, कोई नेटवर्क कनेक्शन नहीं है, कोई खाता नहीं है, और कोई क्लाउड सिंक नहीं है।

- **डेटा जो उपयोग में है:** `release/` फ़ोल्डर में शामिल HTML फ़ाइलें (केवल पढ़ने के लिए) और `localStorage` में `sovereign.autosave` कुंजी के तहत (केवल गेम सेव स्टेट)।
- **डेटा जो उपयोग में नहीं है:** पैकेज निर्देशिका के बाहर किसी भी फ़ाइल सिस्टम तक पहुंच नहीं, किसी भी प्रकार के नेटवर्क अनुरोध नहीं, कोई टेलीमेट्री नहीं, कोई विश्लेषण नहीं, कोई क्रेडेंशियल नहीं।
- **आवश्यक अनुमतियाँ:** ऑपरेटिंग सिस्टम के डिफ़ॉल्ट ब्राउज़र को चलाने की क्षमता, पैकेज की अपनी फ़ाइलों को पढ़ने की क्षमता, ब्राउज़र `localStorage` (वैकल्पिक)।
- **कोई टेलीमेट्री नहीं, कभी नहीं।** सिम्युलेटर की "टेलीमेट्री" सुविधा ब्राउज़र में मौजूद लेज़र से प्राप्त स्थानीय गेम विश्लेषण रिपोर्ट को संदर्भित करती है; ये कभी भी आपके मशीन से बाहर नहीं जाते हैं।

भेद्यता रिपोर्टिंग और पूर्ण सुरक्षा नीति के लिए [`SECURITY.md`](./SECURITY.md) देखें।

## विशेषताएं

- **7 राउंड का एकल खिलाड़ी गेम** दो स्क्रिप्टेड विरोधियों के खिलाफ (डिफ़ॉल्ट रूप से ट्रेजरी/वित्त और व्यापारी/बुनियादी ढांचा; बैच प्ले के लिए निर्माता/उद्योग उपलब्ध)।
- **निर्धारक AI** — प्रत्येक विरोधी के निर्णय दृश्य स्थिति का एक शुद्ध फ़ंक्शन है, जिसके साथ एक लेज़र में कारण दर्ज किया गया है। कोई LLM नहीं, कोई अस्पष्ट जादू नहीं।
- **8 गेम सतहें** — बोर्ड, ट्रेजरी पैनल, एसेट इंस्पेक्टर, इवेंट ड्रावर, कांग्रेस के कार्य, साझा ट्रैक, टर्न लॉग/लेज़र, एंडगेम रिपोर्ट।
- **नीलामी** — अस्वीकृत संपत्तियां प्रोफाइल-आधारित स्क्रिप्टेड बोली के साथ मल्टी-प्लेयर नीलामी में जाती हैं।
- **सेव/लोड** — प्रत्येक टर्न पर `localStorage` में ऑटोसेव, मैनुअल JSON एक्सपोर्ट/इम्पोर्ट, लोड करते समय हैश इंटीग्रिटी चेक, संस्करण-आधारित।
- **रीप्ले** — किसी भी पूर्ण गेम पर पूर्ण स्क्रबर। केवल पढ़ने के लिए। बीज + निर्णय लॉग से पुनर्निर्माण, जिसमें एक हरा इंटीग्रिटी पिल शामिल है।
- **बैच सिमुलेशन** — किसी भी प्रोफाइल त्रिक के खिलाफ 10/50/100 निर्धारक गेम चलाएं, संतुलन विश्लेषण के लिए JSON + HTML रिपोर्ट निर्यात करें।
- **ऐतिहासिक वर्णन** — लेज़र से प्राप्त 25 प्रविष्टियों का पुस्तकालय (डिफ़ॉल्ट रूप से 40-60 शब्द, 150-200 शब्दों का विस्तार, ~300-500 शब्दों का एंडगेम गणराज्य सारांश)। यह कभी भी स्थिति को नहीं बदलता है।
- **पहुंच क्षमता** — पूर्ण कीबोर्ड नेविगेशन, फोकस इंडिकेटर, स्क्रीन रीडर-अर्थपूर्ण लेबल, ट्रैक मान पाठ के रूप में (केवल मार्कर के रूप में नहीं), न्यूनतम 14px बॉडी, मोशन रिडक्शन का समर्थन।

## ज्ञात सीमाएं

- **v1.1.1 एक बीटा संस्करण है।** डिजिटल मोड का सिमुलेशन निदान के विरुद्ध ऑडिट किया गया है, और इन-HTML मानक × 100 बैच में 62/100 अनिवार्य जीत ट्रिगर (अनुमानित 67 के मुकाबले), और 51/33/16 का विजेता विभाजन बिल्कुल अनुमानित के अनुसार है। इसका पूरी तरह से किसी नए मानव खिलाड़ी द्वारा परीक्षण नहीं किया गया है; व्यवहारिक अनुकूलन (यह कि खिलाड़ी अनिवार्य जीत के बारे में जानने के बाद वास्तव में कैसे व्यवहार करते हैं) को मापा नहीं गया है। इसे तब तक वैकल्पिक के रूप में मानें जब तक कि आप स्वयं इसका परीक्षण न कर लें।
- **AI प्रोफाइल अभी तक अनिवार्य जीत के लिए प्रतिस्पर्धा नहीं करते हैं।** वे v0.18 के समान निर्णय फ़ंक्शन का उपयोग करते हैं, जिसका अर्थ है कि वे पूरे गेम में प्रभाव जमा करने के लिए खेलते हैं, न कि जल्दी से 15-IP सीमा तक पहुंचने के लिए। भविष्य के संस्करण में प्रोफाइल निर्णयों को अनिवार्य जागरूकता के लिए समायोजित किया जाएगा। वास्तविक मानव खिलाड़ी अलग तरह से व्यवहार कर सकते हैं।
- **दिवालियापन 12 राउंड में एक महत्वपूर्ण दबाव है।** मानक × 100 में लगभग 7/100 घटनाएं अनिवार्य जीत के साथ होती हैं (बिना अनिवार्य जीत के लगभग 18/100 की तुलना में, क्योंकि गेम जल्दी समाप्त हो जाते हैं)। इसे मानव परीक्षण में देखना सार्थक है।
- **खजाने मंत्रालय / वित्त जानबूझकर सबसे मजबूत है**, लक्षित सीमा के भीतर। यह ऐतिहासिक सिद्धांत से मेल खाता है: सार्वजनिक ऋण + संघीय वित्त, हैमिल्टन के लिए एक प्रमुख आर्थिक उपकरण थे।
- **विफलता घटनाएं (डिफ़ॉल्ट / विद्रोह) ज्यादातर सजावटी हैं।** क्रेडिट संकट 12 राउंड में लगभग 2/100 बार होता है। एस्केलेशन सिस्टम में अधिक समय है, लेकिन फिर भी यह शायद ही कभी डिफ़ॉल्ट या विद्रोह तक पहुंचता है। भविष्य के संस्करण में विफलता स्थिति के दबाव पर पुनर्विचार किया जा सकता है।

## बिल्डिंग और योगदान

```bash
git clone https://github.com/mcp-tool-shop-org/sovereign
cd sovereign
npm install
npm test           # smoke tests
npm run verify     # full verify (smoke + pack-dry-run + CLI flag check)
npm run play       # open the game locally
```

रिलीज़ GitHub एक्शन (`release.yml`) के माध्यम से npm पर प्रकाशित की जाती हैं, जब `v*` टैग पुश किया जाता है, और इसमें Sigstore का प्रमाण शामिल होता है। "main" शाखा ही आधिकारिक स्रोत है।
